# Passo Atual

## Fase Atual

Validação técnica de Unity 3D + MCP oficial.

Ainda não estamos criando o jogo completo. Estamos testando se a Unity realmente reduz atrito no fluxo que mais incomodou no 2D: assets, animações e armas.

## Objetivo do Primeiro Sandbox

Criar uma cena mínima chamada `CombatSandbox` contendo:

- chão simples;
- iluminação básica;
- câmera isométrica/ortográfica;
- player placeholder;
- movimento WASD relativo à câmera;
- dummy de treino;
- ataque básico;
- detecção de hit;
- leitura de console/estado via Unity MCP.

## Critério de Sucesso

O teste é bem-sucedido se:

- o Codex conseguir criar e alterar cena/scripts via Unity MCP;
- o player se mover de forma confortável;
- o ataque básico funcionar contra o dummy;
- a estrutura do projeto continuar organizada;
- o fluxo parecer menos doloroso do que configurar isso manualmente na Unity.

## O Que Não Fazer Ainda

- Não importar Mixamo neste primeiro passo.
- Não criar sistema de armas completo.
- Não criar multiplayer.
- Não criar inventário.
- Não criar UI final.
- Não criar arquitetura grande demais.

Quando esse sandbox estiver funcionando, o próximo teste será importar um humanoide com animações simples.

## Atualizacao 2026-07-08

O passo atual mudou para um sandbox de combate em terceira pessoa.

Critério atualizado:

- camera em perspectiva atras/acima do player;
- mouse controla camera;
- movimento WASD relativo a camera;
- dodge simples;
- ataque basico;
- dummy de treino;
- leitura de console/estado via Unity MCP.

Ainda nao fazer lock-on, multiplayer, inventario, sistema completo de armas ou importacao Mixamo neste passo.
